Cryptomator 1.3.0 RC 5

Pssst! We're about to release a super secret feature: Auto-Unlock on Start. It's the most requested feature in our repository on GitHub and we're excited to give you the chance to be an early tester. It's going to be an experimental feature for 1.3.0 and we'd like to hear your feedback on how it works out for you. :)

Go to your locked vault, press "More Options", and enable "Auto-Unlock on Start". Auto-unlock is currently only available for Windows and macOS because "Save Password" is a requirement.

We're planning to implement "Save Password" and "Auto-Unlock" for Linux in a future release. If you feel at home in Linux and love OSS development, you might be able to help us out here. ;)

This is a prerelease and is only intended for testing. Wait for the stable release for production purposes.

Full release notes are available here.

Download Prerelease Build

We'd like to keep the distribution simple. We're offering binaries for Windows (64/32 Bit), macOS, and Debian (64 Bit).

  • Download Cryptomator 1.3.0-rc5 for Windows (64 Bit)
  • Download Cryptomator 1.3.0-rc5 for Windows (32 Bit)
  • Download Cryptomator 1.3.0-rc5 for macOS
  • Download Cryptomator 1.3.0-rc5 for Debian (64 Bit)

Please use a test vault or ideally a backup/copy of your productive vault. If Cryptomator isn't behaving as expected and the issue isn't reproducible with version 1.2, please let us know.

Other Systems

If your system is not listed, you can also download version 1.3.0-rc5 as a JAR file here, which requires the following installed on your system:

  • Java 8 (min. 8u51, we recommend to use the current version)
  • JCE unlimited strength policy files (needed for 256-bit keys)
